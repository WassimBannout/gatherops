# Portfolio Readiness Checklist

## Reviewer Experience

- [ ] README explains what the project does in the first paragraph.
- [ ] Quick start works from a fresh clone.
- [x] Docker Compose starts dependencies.
- [ ] API docs are easy to find.
- [ ] Example API workflow is included.

## Engineering Signal

- [ ] Architecture diagram exists.
- [ ] ADRs explain major choices.
- [ ] Tests cover meaningful behavior.
- [ ] CI is configured.
- [x] Migrations and schema constraints are clear.
- [ ] Security decisions are documented.

## Polish

- [ ] No stale generated clutter.
- [ ] No hard-coded demo tokens.
- [ ] No obvious spelling mistakes in public docs.
- [ ] Known limitations are honest and specific.
- [ ] Future improvements are realistic.
