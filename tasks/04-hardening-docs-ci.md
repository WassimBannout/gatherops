# Phase 7: Hardening, Docs, CI

## Goal

Turn the working backend into a portfolio-ready project.

## Deliverables

- Complete OpenAPI spec.
- Swagger or Redoc docs route.
- GitHub Actions CI.
- Security checklist completion.
- Code review checklist completion.
- Portfolio README polish.
- Example curl workflow.
- Architecture diagram in README.
- Known limitations section.

## Acceptance Criteria

- Fresh clone instructions work.
- CI runs tests and vet.
- API docs match implemented routes.
- README clearly explains project value.
- Security-sensitive behavior is tested.
- No stale TODOs in production code.
