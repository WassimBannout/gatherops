# Prompt 05: Portfolio Polish

```text
Polish the repository for a hiring-manager and engineer review.

Focus on README, architecture diagrams, setup commands, example API workflow, OpenAPI docs, ADR clarity, and known limitations. Do not add new product features unless needed to make existing behavior understandable.

The README should answer:
- What does this project do?
- Why is it technically interesting?
- How do I run it?
- How do I test it?
- How is it designed?
- What security and reliability decisions were made?
- What would you improve next?

Run documentation checks if available and make sure commands are accurate.
```
